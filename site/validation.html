01 · How to read this ledger

Every figure cited anywhere on this site lives in one row of this ledger: the value, the primary source, how and when it was verified, and which pages use it. If a number on any page isn't traceable to a row here (or to the inline citations of the standalone memos indexed below), it doesn't belong on the site.

Maintenance rule. Numbers change here first. To update a figure: fix its row (value, source, verified date), then propagate to every page in its used on column. A figure with no live row gets removed from the site, not defended.

13 · Per-district figures (Nassau)

The "any Nassau address" action kit and the coverage matrix read one file, assets/districts.json: a source table keyed by the Census school-district name, one row per Nassau district. Every field carries a source_ref (url, vintage, verified date). A field with no verifiable primary source is omitted, never estimated, so a figure only renders per-district when its source resolves. This section documents where each field comes from.

Coverage, in one page. The coverage matrix shows, live, which districts are sourced on which dimension and which are still stubs. It is derived from this same file, so it moves on its own as the table fills. Today: 3 fully modeled pilot districts (Rockville Centre, Long Beach, Locust Valley) and 52 STAR-complete stubs pending the fan-out.

Keying and boundary: districts are keyed by the US Census geocoder school-district name. Nassau mixes unified districts with elementary + central-high-school districts; the geocoder resolves all three layers. Nassau County is Census GEOID 36059; an address that resolves outside it is served the RVC-typical baseline, not a wrong per-district number. Full per-field fetch method: .claude/scratch/districts-provenance.md in the repo.
